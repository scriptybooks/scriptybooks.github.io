ScriptyBooks Open Source: Step One

Apr 15th, 2013

I’ve been intending to open-source the important parts of the ScriptyBooks stack since the beginning, but I never felt right about sharing it as a Rails app. My original bookserver was a simple Node.js app and a command-line build tool, and that seemed like a better path for open source, although I didn’t get far down it before practical concerns on the business side led me to switch to Ruby, Nokogiri, and Rails.

Enter the Jekyll

Octopress + Jekyll is pretty cool stuff. Out of the box, the default blog doesn’t look like much…. but the cmd-line blogging tool is solid. It leverages git and GitHub to the max. Deploying to GitHub Pages is dialed in with its own source branch. Octopress even includes tasks for updating from the original project: Here’s how you update the software, all git based. I should have clued into this stuff a long time ago.
